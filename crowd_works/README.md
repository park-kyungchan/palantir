### 왜 이 프로젝트인가?

크라우드웍스는 **50만+ 크라우드 작업자** 풀을 활용하여 삼성전자, 네이버, 현대차 등
430+ 고객사에 AI 학습 데이터를 공급하는 한국 대표 AI 데이터 기업입니다 (KOSDAQ: 355390).

수리논술 프로젝트는 **한국어 수학 추론 데이터**를 구축하는 핵심 사업으로,
수학 풀이를 LaTeX로 변환하고 JSONL 포맷으로 정제하는 과정의 **품질관리 전문가**가 필요합니다.

수학교육과 전공자로서 이 포지션은 다음 3가지 시너지를 가집니다:

| 시너지 | 설명 |
|--------|------|
| **수식 검수 정확도** | 풀이 과정의 논리적 완결성을 판단하는 전공 역량 |
| **작업자 교육** | 비전문 작업자에게 수학 개념을 효과적으로 설명하는 교육학적 훈련 |
| **오류 패턴 예측** | 학생이 자주 틀리는 유형 ≈ 작업자가 자주 실수하는 유형 |

```
                       현재 ■          필요 □          갭

  수학 내용 이해       ■■■■■■■■■■      □□□□□□□□        갭 없음 (핵심 강점)
  수학 교육 역량       ■■■■■■■■        □□□□□□          갭 없음 (핵심 강점)
  LaTeX 읽기/검수      ■■■■            □□□□□□□□        ████ 중간 (1주)
  JSONL 구조 이해      ■■■■            □□□□□□□□        ████ 중간 (1주)
  LaTeX↔JSONL 이스케이프 ■■             □□□□□□□□        ██████ 높음 (3일 집중)

## 자동화 드릴 훈련 시스템

Claude Code 스킬 체인으로 구축한 **JSONL/LaTeX 역량강화 자동 훈련 시스템**입니다.
사용자가 수학 수식을 JSONL로 변환 → 시스템이 문자 단위로 검증 → 교정 피드백 → 진행도 추적의 루프를 반복합니다.

```

 │                                                                  │

 │   ┌──────────────────┐                                           │

### 스킬 상세

| 스킬 | 도메인 | 역할 | 호출 |
|------|--------|------|------|
| **reference-build** | D0·Foundation | LaTeX/JSONL 명령어 레퍼런스 구축 (Tier1~3) | `/reference-build [수학 도메인]` |
| **challenge-generate** | D1·Drill | 난이도 1~5 렌더링 챌린지 + 트랩 생성 | `/challenge-generate [토픽] [난이도]` |
| **jsonl-validate** | D1·Drill | JSON 파싱, 이스케이프 규칙 R1-R5 검증 | 자동 (드릴 파이프라인) |
| **latex-parse** | D1·Drill | LaTeX 문법 검증 (그룹핑, 연산자, 환경) | 자동 (병렬 실행) |
| **render-evaluate** | D1·Drill | 렌더링 시뮬레이션 + 타겟 비교 | 자동 (결과 합산) |
| **golden-correct** | D1·Drill | 교정본 생성 + 규칙별 마이크로 레슨 | 자동 (드릴 종결) |
| **progress-track** | D2·Eval | 카테고리별 마스터리 추적 + 난이도 추천 | `/progress-track [report\|reset]` |

### JSONL 이스케이프 5대 규칙 (핵심 훈련 대상)

```
┌────────────────────────────────────────────────────────────────┐
│  규칙    │  위반 예시              │  올바른 형태              │
├────────────────────────────────────────────────────────────────┤
│  R1 역슬래시  │  \frac               │  \\frac                │
│  R2 따옴표    │  "정의"              │  \"정의\"              │
│  R3 줄바꿈    │  (실제 개행)          │  \n                    │
│  R4 행구분    │  \\                  │  \\\\                  │
│  R5 중괄호    │  \{                  │  \\{                   │
└────────────────────────────────────────────────────────────────┘

  예시) 수학 수식의 JSONL 변환:

  원본 LaTeX:  \frac{x^{2}}{2}
  JSONL 내부:  \\frac{x^{2}}{2}     ← R1: 역슬래시 이중 이스케이프
```

## 설계 원칙

### SRP (Single Responsibility Principle) 스킬 분리

각 스킬은 **하나의 책임**만 가지며, 명확한 INPUT/OUTPUT으로 체이닝됩니다.
JSONL 구조 검증과 LaTeX 문법 검증을 분리하여 **병렬 실행**이 가능하고,
각 스킬의 독립적인 개선과 테스트가 가능합니다.

### Context Engineering (CE) 정교화

모든 스킬의 L1 Frontmatter는 **1024자 이내**로 라우팅 인텔리전스를 압축하고,
L2 Body는 **Decision Points → Methodology → Failure Handling → Anti-Patterns → Transitions**
구조로 실행 맥락을 완전하게 전달합니다.

### Progressive Deep Dive 조사 방법론

기업과 포지션을 **L1(개요) → L2(맥락) → L3(심층) → L4(갭 분석)** 순서로
점진적으로 깊이를 더해가며 조사하여, 표면적 정보가 아닌 **실질적 준비 전략**을 도출했습니다.

### 1.2 강점 분석 (수학교육과 시너지)

수학교육과 전공은 이 포지션에서 **3가지 핵심 시너지**를 가짐:

1. **수리논술 내용 정확성 판단**
   - 풀이 과정의 논리적 완결성 검증 가능
   - "이 풀이가 맞는가?"를 판단하는 데 비전공자 대비 압도적 우위
   - 수리논술 출제 범위(미적분, 수열, 기하)에 대한 교과과정 이해

2. **작업자 교육 역량**
   - 비전문 작업자에게 수학 개념을 설명하는 데 교육학적 역량 활용
   - "왜 이 LaTeX이 틀린지" 설명 시 수학 교육 방법론 적용 가능
   - 작업 가이드라인 작성 시 교육적 관점 반영

3. **오류 패턴 분석**
   - 학생들이 자주 하는 수학적 실수 패턴에 대한 경험
   - → 작업자가 만드는 LaTeX 오류 패턴과 유사 (괄호 누락, 적분 범위 실수 등)
   - 교육 현장 경험을 데이터 품질 관리에 직접 전이

### 1.3 최대 갭: 운영 역량

기술 스킬(LaTeX/JSONL)은 1-2주 학습으로 충분. **실제 준비해야 할 핵심은 운영 역량:**

- **품질 관리 프레임워크**: KPI 설정, 오류율 추적, 합격/불합격 기준 설계
- **프로젝트 관리**: 일정 관리, 진척률 모니터링, 리스크 대응
- **크라우드 작업자 관리**: 다수의 원격 작업자를 일관된 품질로 관리
- **가이드라인 작성**: 고객사 요구사항 → 작업자용 가이드라인 변환

## 3. 면접에서의 포지셔닝 전략

### "수학교육과 + 데이터 품질 관리"

면접에서 어필할 핵심 메시지:

> "수학교육과에서 학생들의 수학적 사고 과정을 분석하고 교육하는 훈련을 받았습니다.
> 이 역량은 수리논술 데이터의 풀이 정확성 검수와 작업자 교육에 직접 전이됩니다.
> LaTeX/JSONL은 도구이고, 진짜 품질은 수학적 내용의 정확성에서 나옵니다."

### 강점 프레이밍

1. **수학 검수 정확도**: "풀이 과정의 논리적 빈틈을 발견할 수 있습니다"
2. **교육 역량**: "비전문 작업자에게 수학 개념과 LaTeX 규칙을 효과적으로 교육할 수 있습니다"
3. **오류 패턴 예측**: "학생들이 자주 틀리는 유형 = 작업자가 자주 실수하는 유형"
4. **학습 의지**: "LaTeX/JSONL/Python은 이미 기초를 학습했고 계속 역량을 키우고 있습니다"

### 면접 시 반드시 확인할 질문
1. 고용 형태 (정규직 vs 계약직 vs 프리랜서)
2. 프로젝트 기간
3. 작업자 규모 (몇 명 관리?)
4. 고객사 (가능한 범위 내)
5. 팀 구성 (혼자 vs 팀)

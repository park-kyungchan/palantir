## 1. Brainstorming Session Topics

### SESSION 1: LaTeX 수식 리터러시 구축

**목표:** 수식을 보고 LaTeX 코드를 이해하고, 오류를 탐지할 수 있는 수준 달성

#### Topic 1-1: LaTeX 핵심 명령어 레퍼런스 빌드
```
Claude Code에서 실행할 프롬프트:

"수리논술 프로젝트 PA가 알아야 하는 LaTeX 수식 명령어를 체계적으로 정리해줘.
다음 기준으로 분류:
- Tier 1 (매일 마주침): 분수, 거듭제곱, 근호, 그리스문자, 기본 연산
- Tier 2 (자주 마주침): 극한, 미분, 적분, 시그마, 행렬
- Tier 3 (가끔 마주침): 집합, 논리기호, 기하기호, 조합
각 명령어마다 LaTeX코드 → 의미 → 흔한 오타/실수 패턴을 포함.
YAML 형식으로 machine-readable하게 출력해줘."
```

#### Topic 1-2: LaTeX 구문 오류 패턴 분석
```
"수리논술 LaTeX 데이터 검수 시 가장 자주 발생하는 오류 유형을 분석해줘.
카테고리:
1. 구문 오류 (bracket mismatch, 닫는 중괄호 누락 등)
2. 명령어 오류 (오타: \limt → \lim, \sqrtt → \sqrt)
3. 의미 오류 (적분에서 dx 누락, \sum에서 범위 누락)
4. 포맷 오류 (JSONL 안에서의 이스케이프 문제: \\ vs \)
각 유형마다 잘못된 예시 → 올바른 예시 → 탐지 방법을 포함.
Python 스크립트로 자동 탐지할 수 있는 패턴도 같이 만들어줘."
```

#### Topic 1-3: 수리논술 출제 범위별 LaTeX 패턴
```
"한국 대학 수리논술에서 자주 출제되는 수학 영역별로
실제 사용되는 LaTeX 수식 패턴을 정리해줘.
영역: 함수·극한, 미분, 적분, 수열·급수, 확률·통계, 기하·벡터, 행렬.
각 영역마다:
- 대표적인 수식 3-5개 (LaTeX 코드)
- 해당 수식에서 흔히 발생하는 데이터 오류
- 검수 포인트 (무엇을 확인해야 하는지)"
```

### SESSION 2: JSONL 데이터 포맷 마스터

**목표:** JSONL 구조를 이해하고, 수리논술 데이터의 정합성을 검증할 수 있는 수준

#### Topic 2-1: JSONL 기초와 JSON 차이점
```
"JSONL(JSON Lines) 포맷에 대해 다음을 체계적으로 정리해줘:
1. JSON vs JSONL: 구조적 차이, 사용 목적 차이
2. JSONL 규칙: 한 줄 한 객체, 줄바꿈 구분, 배열 아님
3. AI 학습데이터에서 JSONL을 쓰는 이유 (스트리밍, 메모리 효율)
4. 수리논술 프로젝트에서의 JSONL 스키마 설계 예시
   - 문제 데이터: id, topic, question, answer, latex_question, latex_answer, difficulty, metadata
   - 검수 데이터: id, latex_raw, latex_valid, error_type, reviewer, status, timestamp
5. 흔한 JSONL 오류: 후행 쉼표, 배열 형태 작성, 빈 줄, 인코딩 문제
실습용 올바른 예시와 잘못된 예시를 각각 5개씩 만들어줘."
```

#### Topic 2-2: LaTeX in JSONL — 이스케이프 문제
```
"JSONL 안에 LaTeX 수식을 저장할 때 발생하는 이스케이프 문제를 깊이있게 다뤄줘.
핵심 이슈:
- LaTeX의 \frac → JSONL에서는 \\frac (역슬래시 이중 이스케이프)
- \n이 줄바꿈으로 해석되는 문제
- 중괄호 {}가 JSON 구문과 충돌하는 경우
- 실제 데이터에서 자주 깨지는 패턴과 해결법
Python으로 LaTeX 문자열을 JSONL-safe하게 변환하는 유틸리티 함수도 만들어줘."
```

#### Topic 2-3: JSONL 검증 자동화 도구 설계
```
"수리논술 프로젝트 PM/PA가 사용할 JSONL 데이터 검증 도구를 설계해줘.
검증 항목:
1. 구조 검증: JSON 파싱, 필수 필드 존재, 타입 일치
2. LaTeX 검증: 괄호 균형, 유효 명령어, dx 존재
3. 의미 검증: 난이도 값 범위, 토픽 분류 일관성, ID 유니크
4. 통계 리포트: 필드별 완성률, 오류율, 토픽 분포
CLI 도구로 설계하되 Python 기반으로, 결과를 CSV나 JSON으로 출력.
테스트용 샘플 데이터(정상 10개 + 오류 10개)도 같이 생성."
```

## 1. 수리논술이란?

### 1.1 정의

**수리논술 = 수학 기반 서술형 논술 시험.**

한국 대학입시에서 '자연계 논술'의 핵심 파트로, 수학 문제를 풀이 과정까지 논리적으로 서술하는 시험이다. 단순히 정답만 쓰는 것이 아니라, **풀이의 논리적 전개, 수학적 엄밀성, 표현의 정확성**까지 평가한다.

### 1.2 수능과의 차이

| 구분 | 수능 | 수리논술 |
|------|------|---------|
| **형식** | 객관식 (5지선다) | 서술형 (풀이 전개) |
| **시간** | 100분 / 30문항 | 100~150분 / 2~4 대문항 |
| **채점** | 정답 여부만 | 풀이 과정 + 논리성 + 엄밀성 평가 |
| **출제** | 교육과정 엄격 준수 | 교육과정 기반이나 심화 논리 가능 |
| **난이도** | 표준화 | 대학별 편차 큼 (성균관대 쉬움 ~ 인하대 극상) |

### 1.3 출제 범위

**필수:** 수학I, 수학II, 미적분
**선택 (대학별):** 확률과 통계, 기하

**주요 토픽:**
- 함수/극한 (연속성, 극한값 계산)
- 미분 (도함수, 접선, 최대최소, 변화율)
- 적분 (정적분, 부정적분, 넓이, 부피)
- 수열/급수 (점화식, 급수 수렴)
- 확률/통계 (조건부확률, 정규분포)
- 기하/벡터 (이차곡선, 공간벡터)

### 1.4 실시 대학 (주요)

| 대학 | 특징 | 난이도 |
|------|------|--------|
| 연세대(서울) | 논술 100%, 수능 전 시험 | 중~상 |
| 고려대(서울) | 2025학년도 부활 | 중~상 |
| 한양대(서울) | 최저 폐지, 90분 2문제 | 중 |
| 성균관대 | 기출 기반, 접근성 높음 | 하~중 |
| 서강대 | 증명 문제 다수 | 상 |
| 중앙대 | 수학 3문제, 120분 | 중 |
| 인하대 | 의대 수준 문제 | 극상 |

### 1.5 왜 "수리논술"이 데이터로서 가치가 있는가?

수리논술 데이터의 특수성:
1. **풀이 과정이 있다**: 단순 정답이 아닌 step-by-step 논리 전개
2. **수학적 엄밀성**: 수식의 정확한 표현 + 논리적 연결
3. **한국어 + 수식 혼합**: 자연어 설명과 LaTeX 수식이 결합된 멀티모달 텍스트
4. **교육과정 매핑**: 교과서 단원/성취기준과 체계적으로 대응

=> **AI가 "수학적 추론(mathematical reasoning)"을 학습하는 데 최적의 데이터 포맷**

## 2. AI 회사가 수리논술 데이터를 필요로 하는 이유

### 2.1 수학 추론(Math Reasoning)은 AI의 핵심 과제

2024~2026년 AI 업계에서 **수학적 추론(mathematical reasoning)**은 LLM의 핵심 벤치마크이자 연구 주제:

- **GSM8K** (OpenAI): 8,500개 초등수학 문제. Surge AI가 크라우드소싱으로 구축. 2025년에도 월 35만건+ HuggingFace 다운로드.
- **MATH** (UC Berkeley): 12,500개 고등수학 경시대회 문제. 대수/기하/미적분 포함.
- **NuminaMath**: 860,000개 경시대회 수학 문제-풀이 쌍 (최대 공개 수학 데이터셋).
- **OpenMathInstruct-2** (NVIDIA): 14M 문제-풀이 쌍 (Llama 3.1-405B로 생성).
- **Big-Math**: 2025년 공개, RL 학습용 대규모 수학 데이터셋.

### 2.2 한국어 수학 데이터의 공백

**글로벌 수학 데이터셋은 대부분 영어.** 한국어 수학 추론 데이터는 극히 부족:

| 데이터셋 | 규모 | 언어 | 특징 |
|---------|------|------|------|
| AI-Hub 수학 교과 데이터 | 비공개 | 한국어 | 교과서 이미지 -> LaTeX 변환 |
| AI-Hub 수학 문제 풀이과정 데이터 (2024) | 비공개 | 한국어 | 2022 개정교육과정 매핑 |
| jkc-ai/mwp-korean-data | 3,000개 | 한국어 | 한글 서술형 수학문제 JSON |
| 기술과학 문서 요약 | 120,000건 | 한국어 | LaTeX 수식 포함 |

**크라우드웍스의 수리논술 프로젝트는 이 공백을 메우는 작업**으로 추정된다.

### 2.3 수요 시나리오 (추정)

크라우드웍스의 고객사가 다음 목적으로 데이터를 발주했을 가능성:

1. **한국형 MathGPT/수학 AI 튜터 개발**: 네이버, 카카오 등이 한국 교육과정 기반 수학 AI 서비스 개발
2. **LLM 한국어 수학 추론 fine-tuning**: 한국어 chain-of-thought 학습 데이터 필요
3. **교육 AI 서비스**: 수리논술 자동 채점, 풀이 생성, 문제 추천 시스템
4. **정부 데이터바우처 사업**: 교육 분야 AI 학습 데이터 구축 과제

## 3. 수리논술 AI 학습 데이터의 구조

### 3.1 데이터 형태 (추정)

수리논술 프로젝트의 산출물은 다음과 같은 구조의 JSONL 데이터일 가능성이 높다:

```jsonl
{"id": "SN-2026-0001", "topic": "미분", "subtopic": "접선의방정식", "difficulty": 3, "question": "곡선 y = x^3 - 3x 위의 점 (1, -2)에서의 접선의 방정식을 구하시오.", "question_latex": "곡선 $y = x^3 - 3x$ 위의 점 $(1, -2)$에서의 접선의 방정식을 구하시오.", "solution": "f(x) = x^3 - 3x로 놓으면...", "solution_latex": "$f(x) = x^3 - 3x$로 놓으면\n$f'(x) = 3x^2 - 3$\n$f'(1) = 3(1)^2 - 3 = 0$\n따라서 접선의 기울기는 $0$이고,\n접선의 방정식은 $y = -2$", "answer": "y = -2", "answer_latex": "$y = -2$", "source": "수리논술 기출 변형", "curriculum": "수학II > 미분 > 도함수의 활용", "metadata": {"university_level": "중위권", "year": 2026, "reviewer": "R-042"}}
```

### 3.2 LaTeX의 역할

**LaTeX은 수식을 기계가 읽을 수 있는 형태로 표현하는 표준 언어.**

수리논술 데이터에서 LaTeX이 필수인 이유:
1. **수식의 정확한 표현**: `\frac{d}{dx}(x^3) = 3x^2` -- 분수, 미분, 지수가 정확히 인코딩됨
2. **AI 학습 호환성**: LLM이 수식을 이해하려면 텍스트 기반 표현이 필요 (이미지 X)
3. **렌더링 가능**: MathJax, KaTeX로 예쁜 수식으로 변환 가능 (검증 용이)
4. **표준화**: 전 세계 학술/교육 커뮤니티의 수식 표기 표준

**수리논술 프로젝트 PM이 모니터링해야 할 LaTeX 이슈:**
- 괄호 불일치: `\frac{1}{2` (닫는 중괄호 누락)
- 명령어 오타: `\limt` -> `\lim`, `\sqrtt` -> `\sqrt`
- 적분 dx 누락: `\int_0^1 x^2` -> `\int_0^1 x^2 \, dx`
- 환경 미닫힘: `\begin{cases}` 있는데 `\end{cases}` 없음

### 3.3 JSONL의 역할

**JSONL(JSON Lines)은 AI 학습 데이터의 사실상 표준 포맷.**

| 특성 | JSON | JSONL |
|------|------|-------|
| 구조 | 하나의 배열 객체 | 한 줄에 하나의 JSON 객체 |
| 메모리 | 전체 로딩 필요 | 스트리밍 처리 가능 |
| 대용량 | 비효율 | 효율적 (줄 단위 처리) |
| 병렬 | 어려움 | 줄 단위 병렬 처리 |
| Git diff | 불편 | 줄 단위 변경 추적 |

**JSONL + LaTeX 결합 시 이스케이프 이슈:**
```
LaTeX 원본: \frac{1}{2}
JSONL 내부: "\\frac{1}{2}"  (역슬래시 이중 이스케이프)

LaTeX 원본: 줄바꿈이 있는 풀이
JSONL 내부: "첫째줄\\n둘째줄"  (\n이 줄바꿈으로 해석되면 JSONL 구조 파괴)
```

이 이스케이프 문제가 PM이 가장 자주 마주하는 기술적 이슈 중 하나.

## 4. 데이터 구축 프로젝트 일반 워크플로

### 4.1 AI 학습 데이터 구축 표준 프로세스 (AI-Hub 기준)

```
[1단계] 데이터 수집 (Raw Data)
    |
[2단계] 가공/라벨링 (Annotation)
    |-- 작업 가이드라인 배포
    |-- 작업자 교육
    |-- 크라우드소싱 작업
    |
[3단계] 품질 검수 (QA/QC)
    |-- 1차: 자동 검증 (스크립트)
    |-- 2차: 검수자 수동 검수
    |-- 3차: PM/PL 최종 확인
    |
[4단계] 데이터셋 구성
    |-- 학습데이터 (50%)
    |-- 검증데이터 (30%)
    |-- 평가데이터 (20%)
    |
[5단계] 납품 및 AI 모델 검증
```

### 4.2 수리논술 프로젝트 특화 워크플로 (추정)

```
[기획] 고객사 요구사항 분석
  -> 데이터 스키마 설계 (JSONL 필드 정의)
  -> 작업 가이드라인 작성 (LaTeX 작성 규칙 포함)
        |
[작업] 작업자 모집 + 교육
  -> 수리논술 문제/풀이 LaTeX 변환 작업
  -> 실시간 진척률 모니터링
        |
[검수] 자동 검증 (LaTeX 구문, JSONL 구조)
  -> 수동 검수 (수학적 정확성, 논리적 완결성)
  -> 불량 데이터 반려 + 재작업
        |
[납품] 최종 JSONL 파일 패키징
  -> 통계 리포트 (토픽별 분포, 난이도별 분포, 오류율)
  -> 고객사 전달
```

**PM/PL의 관여 지점:** 기획 전체 + 작업 모니터링 + 검수 관리 + 납품 QA

### 5.3 크라우드웍스 수리논술 프로젝트의 위치

기존 한국어 수학 데이터셋은 대부분:
- 이미지 기반 (OCR 활용) 또는
- 소규모 (3,000개) 또는
- 교육과정 범위 한정 (초등~고등 교과서)

**수리논술 데이터의 차별점:**
- **서술형 풀이 포함** (chain-of-thought)
- **대학 수준 난이도** (고교 최상위 ~ 대학 교양)
- **LaTeX 네이티브** (이미지가 아닌 텍스트 기반)
- **체계적 메타데이터** (토픽/난이도/교육과정 매핑)

=> 한국어 수학 추론 AI의 "프리미엄 fine-tuning 데이터"에 해당

## 6. 핵심 인사이트 (Key Insights)

### 6.1 프로젝트 맥락 이해

1. **이 프로젝트는 "데이터 노가다"가 아니다**: 수학적 정확성 + LaTeX 기술 + AI 데이터 포맷 이해가 결합된 전문 작업
2. **고객사는 수학 AI를 만들려는 기업**: 네이버/카카오/에듀테크 기업이 수학 AI 튜터나 한국어 수학 LLM을 개발 중일 가능성
3. **데이터 품질이 곧 AI 성능**: 크라우드웍스의 99% 품질 보장 약속 -- PM이 이 품질을 관리해야 함

### 6.2 PM에게 필요한 도메인 지식

| 영역 | 필요 수준 | 근거 |
|------|----------|------|
| **수리논술 수학 내용** | 중상 | 문제/풀이의 정확성 판단 가능해야 함 |
| **LaTeX 문법** | 중 | 구문 오류 탐지, 작업자 교육 가능 수준 |
| **JSONL 구조** | 중 | 데이터 정합성 점검, 스키마 이해 |
| **데이터 품질관리** | 상 | PM의 핵심 업무 |
| **크라우드소싱 운영** | 중상 | 작업자 관리, 진척률 관리 |

### 6.3 산업 트렌드와 연결

- **2025~2026년**: 수학 추론은 LLM의 가장 중요한 벤치마크 중 하나. GSM8K, MATH 점수가 모델 성능의 핵심 지표.
- **한국어 데이터 공백**: 영어 수학 데이터는 1400만개 이상이 공개되어 있지만, 한국어 수학 추론 데이터는 극히 부족.
- **크라우드웍스의 전략적 가치**: 한국어 수학 데이터를 대규모로 구축할 수 있는 인프라(50만 작업자 + 1000건 프로젝트 경험)를 보유한 유일한 기업.

  2. 작동 흐름

  사용자 프롬프트 입력
          ↓
  CLAUDE.md 자동 로드 (system-reminder로 주입)
          ↓
  ODA 프로토콜 적용:
  ├── Schema-First: ObjectTypes 기준
  ├── Action-Only: 등록된 Actions만 허용
  ├── Audit-First: 모든 작업 로깅
  └── Zero-Trust: 파일 검증 필수
          ↓
  3-Stage Protocol 실행 (A→B→C)

  3. 계층 구조
  ┌─────────┬─────────────────────────┬───────────────────────────────┐
  │ 레이어  │          경로           │             역할              │
  ├─────────┼─────────────────────────┼───────────────────────────────┤
  │ Layer 1 │ .claude/CLAUDE.md       │ 핵심 Identity, 오케스트레이션 │
  ├─────────┼─────────────────────────┼───────────────────────────────┤
  │ Layer 2 │ .claude/references/*.md │ 서브에이전트 상세             │
  ├─────────┼─────────────────────────┼───────────────────────────────┤
  │ Layer 3 │ .claude/skills/*.md     │ 워크플로우 실행               │
  ├─────────┼─────────────────────────┼───────────────────────────────┤
  │ Layer 4 │ .claude/agents/*.md     │ 특화 동작                     │
  └─────────┴─────────────────────────┴───────────────────────────────┘
  4. 항상 적용되는 규칙

  - TodoWrite 사용하여 작업 추적
  - 3-Stage Protocol (SCAN → TRACE → VERIFY)
  - Evidence 수집 (files_viewed 필수)
  - Task 위임 (복잡한 작업은 서브에이전트로)

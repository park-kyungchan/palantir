2. Mathpix 출력 형식 기준
"Mathpix 결과(LaTeX/텍스트)"라고 했는데, Mathpix는 여러 출력 형식을 지원합니다:

mmd (Mathpix Markdown)
latex
text
data (JSON 구조화)
→ 어떤 형식을 기준으로 text_spec 스키마를 설계할까요?

3. 수학 문제의 타겟 수준/범위
"전 범위"라고 했지만:

한국 수능/내신 기준인가요?
초중고 전체인가요, 고등 수학만인가요?
대학 수학(미적분학, 선형대수 등)도 포함인가요?
→ 예시 JSON 작성 및 도형 복잡도 설계에 영향을 줍니다.

4. Regeneration 출력 형식 타겟
"파라메트릭 표현"을 만든다고 했는데, 최종 렌더링 타겟이:

TikZ (LaTeX)
SVG
Desmos/GeoGebra API 호환
자체 JSON → 별도 렌더러
→ 어떤 쪽으로 설계할까요?

확인하지 않아도 되는 부분
"최신 Claude 모델" → 공식 문서 검색으로 자체 확정 가능
"DOCX generation spec" → 구현 코드 없이 명세만 작성하라고 명시됨
Human-in-the-loop UI → 기능 요구사항 수준이라고 명시됨

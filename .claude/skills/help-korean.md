# Claude Code 한국어 도움말

## 실시간 문서 검색 (NEW)

이 도움말 스킬은 **Task 도구**를 통해 최신 정보를 실시간으로 검색할 수 있습니다.

### 실시간 검색 가능한 정보

| 질문 유형 | 검색 방법 | 예시 |
|----------|----------|------|
| 최신 CLI 기능 | Task → Explore | "새로 추가된 기능 뭐 있어?" |
| MCP 서버 사용법 | Task → 문서 검색 | "GitHub MCP 어떻게 써?" |
| 프로젝트 맞춤 도움 | Task → 코드 분석 | "이 프로젝트에서 어떤 명령어 써?" |

### 사용 예시
```
"최신 Claude Code 기능 알려줘"
→ Task 도구로 claude-code-guide 문서 검색

"MCP 서버 설정 방법 자세히"
→ Task 도구로 MCP 문서 및 예제 검색

"이 프로젝트에 맞는 /audit 사용법"
→ Task 도구로 현재 프로젝트 분석 후 맞춤 안내
```

## 기본 사용법

### 대화형 명령
Claude Code는 자연어를 이해합니다. 그냥 한국어로 말하세요:

```
"이 코드가 뭐하는 건지 설명해줘"
"버그를 찾아서 고쳐줘"
"테스트 코드 작성해줘"
"이 함수 리팩토링해줘"
"성능 개선해줘"
```

### 슬래시 명령어
특별한 기능을 빠르게 실행:

| 명령어 | 설명 | 예시 |
|--------|------|------|
| `/init` | 워크스페이스 초기화 | `/init` |
| `/audit` | 코드 품질 검사 | `/audit hwpx` |
| `/plan` | 계획 모드 진입 | `/plan` |
| `/ask` | 도움 요청 | `/ask 이거 어떻게 해?` |
| `/compact` | 대화 정리 | `/compact` |
| `/mcp` | MCP 서버 상태 | `/mcp` |
| `/resume` | 이전 세션 복구 | `/resume` |

### 키보드 단축키

| 단축키 | 기능 |
|--------|------|
| `Ctrl+C` | 현재 작업 취소 |
| `Ctrl+L` | 화면 지우기 |
| `Shift+Tab` | Plan Mode 전환 (두 번) |
| `Esc Esc` | 이전 상태로 되돌리기 |
| `Ctrl+B` | 백그라운드로 전환 |
| `Alt+T` | Extended Thinking 토글 |

## 주요 기능

### 1. Plan Mode (계획 모드)
코드 수정 전 계획을 먼저 보여주는 안전 모드

**활성화:** `Shift+Tab` 두 번
**사용 시점:**
- 여러 파일 수정 필요할 때
- 대규모 리팩토링
- 실수가 걱정될 때

### 2. 체크포인트 & 되감기
모든 변경사항 자동 저장, 언제든 복구 가능

**되돌리기:** `Esc` 두 번 또는 `/rewind`

### 3. Background Tasks
긴 작업을 백그라운드에서 실행

**전환:** `Ctrl+B`
**확인:** "백그라운드 작업 상태 알려줘"

### 4. MCP 서버 (외부 연동)
GitHub, 데이터베이스 등 외부 서비스 연동

**확인:** `/mcp`
**사용:** "GitHub에서 이슈 목록 가져와줘"

### 5. Extended Thinking
복잡한 문제에 더 깊이 생각

**활성화:** `Alt+T` 또는 프롬프트에 "ultrathink" 포함
**사용 시점:** 복잡한 알고리즘, 아키텍처 설계

### 6. Task 도구를 통한 위임 (NEW)
복잡한 작업을 서브에이전트에게 위임

**사용 시점:**
- 넓은 범위의 코드 탐색 필요
- 최신 문서 검색 필요
- 병렬 분석 필요

**위임 가능한 에이전트:**
| 에이전트 | 용도 |
|----------|------|
| Explore | 코드베이스 빠른 탐색 |
| Plan | 구현 계획 수립 |
| general-purpose | 복잡한 다단계 작업 |

## 상황별 가이드

### "코드를 이해하고 싶어요"
```
이 파일 분석해줘
이 함수가 뭐하는 건지 설명해줘
프로젝트 구조 설명해줘
/audit
```

### "버그를 고치고 싶어요"
```
이 에러 왜 나는지 알려줘
이 버그 고쳐줘
테스트 실패 원인 찾아줘
```

### "새 기능을 추가하고 싶어요"
```
Shift+Tab 두 번 (Plan Mode 진입)
[원하는 기능 설명]
[계획 검토 후 승인]
```

### "리팩토링하고 싶어요"
```
이 코드 리팩토링해줘
클린 코드로 개선해줘
성능 최적화해줘
```

### "문서화하고 싶어요"
```
이 함수에 docstring 추가해줘
README 작성해줘
API 문서 생성해줘
```

### "최신 기능이 궁금해요" (NEW)
```
Claude Code 새 기능 뭐 있어?
MCP 서버 목록 보여줘
Extended Thinking 어떻게 써?
```

## 팁 & 트릭

### 효과적인 프롬프트 작성
1. **구체적으로**: "이 코드 고쳐줘" → "line 45의 null 체크 추가해줘"
2. **맥락 제공**: "사용자 인증 관련 코드인데..."
3. **원하는 결과 명시**: "테스트도 같이 작성해줘"

### 실수 방지
1. 중요한 변경 전 Plan Mode 사용
2. `Esc Esc`로 언제든 되돌리기
3. 큰 작업은 단계별로 진행

### 비용 절약
1. 간단한 작업은 haiku 모델 사용
2. `/compact`로 대화 정리
3. 불필요한 파일은 `.claudeignore`에 추가

## 문제 해결

| 문제 | 해결 |
|------|------|
| "명령어가 안 돼요" | `/init` 다시 실행 |
| "너무 느려요" | `/compact` 후 다시 시도 |
| "실수했어요" | `Esc Esc`로 되돌리기 |
| "모르겠어요" | `/ask 도와줘` 입력 |
| "최신 정보 필요해요" | Task 도구로 문서 검색 요청 |
